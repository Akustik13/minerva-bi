⭐ Позначки та нотатки до замовлень

💡
Доступно в обох режимах перегляду: картки та таблиця. Зміни відбуваються без перезавантаження сторінки — через AJAX.
⭐

Позначити важливе

Кнопка ☆ у картці або колонці таблиці. Клік — стає ⭐ золотою. Картка виділяється золотою лівою рамкою. Фільтр: кнопка ⭐ Важливі у панелі швидких фільтрів або ?flagged=1.

📝

Внутрішня нотатка

Кнопка 📝 у підвалі картки або колонці таблиці. Натисніть — розкривається поле вводу. Enter або кнопка «💾 Зберегти» — збережено. Esc — закрити без збереження. Нотатка видима в картці та таблиці (до 22 символів + «…»).

Крок 2 — Запустіть стратегію

1

Відкрийте CRM → Стратегії → + Додати

Оберіть клієнта (пошук за ім'ям або компанією) і шаблон зі списку.

2

Збережіть — перший крок з'явиться автоматично

Minerva відразу створює перший крок із шаблону (наприклад, "Email — повернення"). Нічого більше заповнювати не потрібно.

3

Натисніть "🗺 Відкрити canvas"

Відкриється сторінка з переліком кроків та SVG-схемою. Перший крок виділений синьою рамкою — це поточний крок.

Крок 3 — Щоденна робота: фіксуйте результати

📋
Щодня відкривайте Стратегії → фільтр "Активна" і обробляйте клієнтів, у яких настав час наступного кроку.
1

Відкрийте canvas стратегії

На сторінці стратегії → кнопка "🗺 Відкрити canvas". Або натисніть іконку в списку стратегій.

2

Знайдіть поточний крок (синя рамка)

У режимі ☰ Список — поточний крок має форму для заповнення результату. У режимі ⬡ Canvas — синя крапка в правому куті вузла.

3

Виконайте дію і зафіксуйте результат

Наприклад, надіслали email → оберіть ✅ Виконано (+) і напишіть нотатку. Натисніть "Зберегти".

4

Minerva переходить до наступного кроку автоматично

Сторінка оновлюється. Наступний крок стає активним — або стратегія завершується, якщо кроків більше немає.

Практичний приклад: Реактивація клієнта

📧

День 0 — Email повернення

Надсилаєте email з персональною пропозицією. Фіксуєте: ✅ Виконано (+), нотатка "Надіслано email зі знижкою 10%".

🔀

День 5 — Рішення: відповідь на email?

Якщо клієнт відповів → ✅ Виконано (+) → переходимо на Дзвінок.
Якщо не відповів → 🔇 Без відповіді → переходимо на Паузу 90 днів.

📞

День 8 — Дзвінок (якщо відповів)

Телефонуєте, уточнюєте потреби. Фіксуєте деталі в нотатках.

🔀

День 9 — Рішення: зацікавлений?

ТАК → стратегія завершується успішно ✅.
НІ → Пауза 90 днів → повернетесь пізніше.

Мобільний режим та PWA

ℹ️
PWA (Progressive Web App) — технологія що перетворює веб-сайт на нативний додаток: власна іконка, повноекранний режим без адресного рядка, кешування статики, офлайн-заглушка.

📲 Встановити Minerva на телефон

🤖

Android (Chrome)

1. Відкрийте Minerva в Chrome
2. З'явиться банер "Встановити Minerva" внизу екрану — натисніть
або: меню ⋮ → "Додати на головний екран"

Minerva відкриватиметься як окремий додаток без адресного рядка.

🍎

iPhone / iPad (Safari)

1. Відкрийте Minerva в Safari (тільки Safari на iOS)
2. Кнопка "Поділитись" (□↑) внизу екрану
3. Прокрутіть список → "На початковий екран"
4. Підтвердіть назву → "Додати"

💻

Desktop (Chrome / Edge)

1. Відкрийте Minerva в Chrome або Edge
2. Іконка ⊞ в адресному рядку праворуч або банер в нижньому правому куті
3. Натисніть "Встановити"

🌐 Офлайн-режим

⚠️
Minerva працює з живими даними, тому повноцінна робота потребує з'єднання. Офлайн Service Worker лише відображає заглушку і повертає кешовані сторінки де можливо.
✅

Доступно без інтернету

Статичні файли (CSS, JS, іконки) — Cache First, завжди зі кешу.
Сторінки що відкривались нещодавно — можуть бути доступні в режимі читання.

❌

Потребує інтернету

Збереження змін, нові дані, синхронізація, API-запити.
При відсутності мережі — сторінка "🏛️ Minerva недоступна" з кнопкою "Спробувати знову".

🔄 Оновлення після git pull

⚠️
Якщо оновлення містить зміни в static/ (CSS, JS, іконки) — необхідно запустити collectstatic, інакше нові файли не з'являться у production через WhiteNoise.
1

Оновити через Minerva (рекомендовано)

Система → Резервне копіювання → блок "🔄 Оновлення системи" → кнопки "🔄 Перевірити" → "⬇️ Оновити" → "🔁 Перезапустити веб"

2

Або вручну через SSH

git pull
docker-compose exec web python manage.py collectstatic --noinput
docker-compose restart web

3

Скинути кеш Service Worker (якщо щось злипло)

Chrome DevTools → Application → Service Workers → "Unregister" → перезавантажити сторінку.
Або відкрити в режимі інкогніто.
