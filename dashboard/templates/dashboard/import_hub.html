📥 Імпорт даних

🛒 Замовлення (Sales)
Імпорт замовлень з Excel — 3-кроковий wizard з маппінгом колонок, dry-run та auto-прив'язкою клієнтів.
⬆️ Відкрити wizard 📄 Шаблон .xlsx
▶ Колонки файлу (24 поля)
order_number ✱ source ✱ order_date status client email phone addr_street addr_city addr_zip addr_country shipping_courier tracking_number shipping_deadline shipped_at currency shipping_cost document_type affects_stock sku_raw (рядки) qty (рядки) unit_price (рядки)
Конфлікти: при збігу source + order_number — пропустити або оновити (вибір у wizard).
Dry-run: перший запуск без запису в БД — побачиш результат без ризику.
🔄 Авто-імпорт по розкладу
Профілі імпорту з папки або URL (Google Sheets). Cron запускає автоматично — замовлення, товари, прихід, коригування.
📥 Імпортувати 📋 Профілі
Google Sheets: вкажіть URL ...export?format=csv як джерело.
Дедуплікація: SHA256 файлу — один і той самий файл не буде імпортовано двічі.
📦 Товари та склад (Inventory)
Імпорт товарів і початкових залишків. Якщо SKU вже існує — оновлення полів за вибором.
⬆️ Відкрити wizard 📄 Шаблон .xlsx
▶ Колонки файлу (14 полів)
sku ✱ name category manufacturer purchase_price sale_price reorder_point lead_time_days initial_stock unit_type hs_code country_of_origin net_weight_g is_active
initial_stock: при вказанні → створює транзакцію ADJUSTMENT.
category: slug категорії (acoustic-panels, accessories, raw-materials…).

📋 Правила імпорту

Формат файлу: .xlsx (Excel 2007+). Перший рядок — заголовки. Wizard сам визначає листи і колонки.
Дати: YYYY-MM-DD або DD.MM.YYYY. Числа — крапка або кома як роздільник.
Клієнти: при імпорті замовлень — auto-створення Customer по email+name (SHA256 ключ).
Джерела: поле source має відповідати slug в розділі Продажі → Джерела.
⚠️ Щось пішло не так?
Якщо імпорт дав невірні дані — скинь транзакційні дані і починай заново. Товари, категорії, джерела і налаштування — залишаться.
